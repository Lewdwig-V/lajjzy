# M3a: Omnibar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace fuzzy-find with a unified omnibar that does live fuzzy matching while typing and evaluates jj revset expressions on Enter.

**Architecture:** The omnibar is a drop-in replacement for `Modal::FuzzyFind`. Typing behavior is identical (client-side fuzzy match). Enter triggers a new `Effect::EvalRevset` that calls `backend.load_graph(Some(&query))`. Success refilters the graph with a status bar breadcrumb; failure falls back to fuzzy jump. `RepoBackend::load_graph` gains a `revset: Option<&str>` parameter.

**Tech Stack:** Rust 1.85+, ratatui 0.30, crossterm 0.29, jj CLI 0.39.0

**Spec:** `docs/superpowers/specs/2026-03-21-m3a-omnibar-design.md`

- [ ] **Step 4: Implement `RevsetLoaded` handler**

Replace the placeholder:

```rust
Action::RevsetLoaded { query, generation, result } => {
    state.omnibar_fallback_idx = None;
    match result {
        Ok(new_graph) => {
            if new_graph.node_indices().is_empty() {
                state.status_message = Some(format!("No changes match: {query}"));
            } else {
                state.active_revset = Some(query);
                let nested = dispatch(state, Action::GraphLoaded {
                    generation, result: Ok(new_graph),
                });
                debug_assert!(nested.is_empty());
            }
        }
        Err(_) => {
            if let Some(idx) = state.omnibar_fallback_idx.take() {
                state.cursor = idx;
                state.reset_detail();
            }
        }
    }
}
```

Note: `omnibar_fallback_idx` is set to `None` at the top, but the `Err` branch also calls `.take()` on it. Since it was already cleared at the top, the `.take()` on the Err path will always be `None`. This is a bug in the spec — the fallback should NOT be cleared at the top if we want the Err path to use it. Fix: only clear it on the Ok path.

```rust
Action::RevsetLoaded { query, generation, result } => {
    // Reject stale revset results (same check as GraphLoaded)
    if generation < state.graph_generation {
        return vec![];
    }

    match result {
        Ok(new_graph) => {
            state.omnibar_fallback_idx = None; // consumed
            if new_graph.node_indices().is_empty() {
                state.status_message = Some(format!("No changes match: {query}"));
            } else {
                state.active_revset = Some(query);
                let nested = dispatch(state, Action::GraphLoaded {
                    generation, result: Ok(new_graph),
                });
                debug_assert!(nested.is_empty());
            }
        }
        Err(_) => {
            if let Some(idx) = state.omnibar_fallback_idx.take() {
                state.cursor = idx;
                state.reset_detail();
            }
        }
    }
}
```

In a real jj repo:
1. `/` — omnibar opens empty, fuzzy results show all changes
2. Type a description fragment — results narrow (live fuzzy match)
3. `Enter` on a fuzzy match — jumps to that change (same as before)
4. `/` then type `@` then `Enter` — graph refilters to show only working copy, breadcrumb shows `revset: @`
5. `/` — omnibar pre-fills with `@`, can edit or clear
6. Backspace to clear, `Enter` — revset cleared, full graph restored
7. `/` then type `mine()` then `Enter` — refilters to user's changes
8. `d` to abandon a change — graph refreshes with revset still active
9. `R` to refresh — revset still active
10. `/` then type garbage then `Enter` — falls back to fuzzy jump, no refilter
